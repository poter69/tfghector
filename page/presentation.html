1. Introducción y Objetivos

Este proyecto no es solo un ejercicio académico; nace de una necesidad muy real. Tras una etapa profesional anterior, fui testigo de las dificultades que enfrentaban mis compañeros para realizar tareas de remaquetación de equipos de forma eficiente y consistente. Esa experiencia sembró la semilla de este TFG: buscar, implementar y dominar herramientas que realmente solucionen estos desafíos en el día a día de un administrador de sistemas.

Motivación

Optimizar el despliegue masivo de SO, reduciendo tiempo y errores, para entornos educativos y PYMES.

Objetivos

  • Instalar y configurar FOG y WDS.
  • Documentar captura y despliegue (Win/Linux).
  • Comparar funcionalidad, flexibilidad y coste.
  • Proponer FOG como solución principal, demostrando Snapins.

3. FOG Project: Solución Open Source

¿Qué es FOG Project?

Solución libre (GPLv3) para clonación y despliegue de imágenes de SO (Win, Linux, macOS) vía red (PXE, TFTP, NFS). Altamente personalizable y gratuita.

Puesta en Marcha

  • VM Ubuntu Server 24.04.2 LTS.
  • Instalación FOG v1.5.10.1650.
  • Red interna (172.16.0.100) con DHCP propio para PXE.
  • Gestión vía interfaz web.

* Anexos II y III.

3.1 FOG: Proceso y Snapins

Proceso Clave

  1. Preparar Imagen Maestra: SO base + software. Sysprep (Win) / Script limpieza (Linux). Instalar FOG Client.
  2. Capturar Imagen: Registrar maestro en FOG, crear tarea de captura, arrancar por PXE.
  3. Desplegar Imagen: Registrar cliente, asignar imagen, crear tarea de despliegue, arrancar por PXE.

* Memoria Cap. 10, Anexo VIII.

Snapins: Automatización Avanzada

Scripts/ejecutables para tareas post-instalación.

  • Instalación Office 365.
  • Cambio de nombre de equipo.
  • Creación/segurización de usuarios.

* Anexos VII y VIII.

4. Windows Deployment Services (WDS)

¿Qué es WDS?

Solución Microsoft para despliegue de SO Windows vía red (PXE, imágenes WIM). Integrado con AD, DHCP, DNS.

Experiencia y Limitaciones

  • Incompatibilidad: Problemas con `boot.wim` modernos en Win Server 2025.
  • Obsolescencia: Microsoft lo descontinúa por Intune.
  • Menor flexibilidad: Automatización post-despliegue limitada.
  • Proceso más manual y rígido.

* Anexos IV, V, IX.

6. Conclusiones y Futuro

FOG Project: Superior

Recomendado por flexibilidad, multiplataforma, Snapins, sin coste y comunidad activa. Flujo completo y eficiente demostrado.

WDS: Limitado

Incompatibilidades en Win Server 2025 y obsolescencia. Uso futuro desaconsejable para nuevos proyectos.

Aprendizaje y Líneas Futuras (FOG)

  • Compatibilidad Secure Boot, integración AD, despliegue multinodo, VPN.
  • Mejora de gestión de Snapins y documentación.
